idea 远程调试记录

场景：

本地windows，开发机centos，涉及到的数据库为开发机内网ip，开发机端口有限制，不能在idea中直连，搜到的教程都是在idea中直接填写开发机ip和端口，所以都没有办法直接使用。

配置：

开发机：

tomcat：

配置 $tomcat_home/conf/server.xml ：配置tomcat启动端口，如8080。

配置$tomcat_home/bin/catalina.sh ：在

上边加上

目的是让tomcat启动时可以被本地debug attach上来。

nginx：

配置端口转发：（由于在本地无法直接访问8080端口，使用nginx做反向代理，在本地请求api.xxx.com转发到tomcat对应端口，需要在windows上配置host，这个稍后说。）

注意这里的端口是tomcat的项目监听端口，8080

启动服务

启动tomcat： $tomcat_home/bin/startup.sh

查看 $tomcat_home/logs/catalina.out 日志确认启动成功。

重启nginx nginx -s reload

本地机器（windows）：

这块的配置在网上没找到比较好的教程，写的都是直接访问开发机的端口，这里采用xshell隧道把开发机的端口映射到本地机器然后通过idea代理访问本地端口来实现调试的。

xshell配置：

打开属性 —-> 连接 —-> 隧道 —-> 添加

类型（方向） ： Local(Outgoing)

源主机： localhost

侦听端口：随便选一个本地没有被占用的端口（如46666）

目标主机：开发机公网ip

目标端口：上边配置的5005端口

然后开启一个xshell页面，把这个页面的查看 —> 隧道窗格选项勾选出来，可以看到窗口下边转移规则状态为打开证明已经开启 。

idea配置：

这一步就是很多博客都写的了，不同的是配置host为127.0.0.1 端口为上边设置的46666。

配置方法：

打开run /debug configurations 窗口，点击加号，选remote

填写： host :127.0.0.1 port:46666

开始调试：

到这一步几乎所有博客也是说“开始尽情调试吧”，还加上了一个断点截图，对于刚开始接触开发的同学来说还是有点懵，加了断点咋就执行了。

这块需要把项目打成war包发布到开发机上，在本地项目和远程项目一致时，本地打了断点，浏览器访问远程的项目到这一步就会停在这了，这才可以尽情调试。

步骤： mvn clean package —> 发布到开发机 —-> 重启tomcat —->打断点 —–> 点debug

连接成功后，访问对应的接口就可以了。
